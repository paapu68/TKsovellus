Running Web2Py on Heroku

I have been using the Web2PY Framework for Python for almost an year now and I like it for all the abstraction it provides, allowing me to run my app virtually anywhere, without changing the code.

So far we’ve been using Google AppEngine for our projects, but I decided to try Heroku with General Knowitall Knowledge's handler, that we plan to use for managing user feedback and serving new questions over-the-air.

There is plenty of documentation on running Django with Heroku, but very little for Web2PY. So, I decided I’d write down the steps I took to get my Web2Py app running on Heroku:

  1. Use the Source-Code version of Web2Py and set your app up locally
  2. Get a Heroku account and install the Heroku client for your OS, following their documentation
  3. Next you’ll need to add a Procfile in the base directory of your Web2Py installation with the following command: web: python anyserver.py -s gunicorn -i 0.0.0.0 -p $PORT
  4. And then you’ll need to add a requirements.txt file, which lists the python modules that Heroku should install with PIP, it should contain:  
    web2py
gunicorn
psycopg2
    and any other modules you may need.
  5. Now you have to set your app to use Heroku’s PostgreSQL, for this, open your db.py model and replace the sqlite:// URL with the one in Heroku’s DATABASE_URL environment variable; here is what we did: 
    import os
...
if not request.env.web2py_runtime_gae:
    ## if NOT running on Google App Engine use SQLite or other DB
    try: db = DAL(os.environ.get('DATABASE_URL'))
    except: db = DAL('sqlite://storage.sqlite')
...
  6. Now create the Heroku app and deploy it as you normally would, following the rest of their documentation
